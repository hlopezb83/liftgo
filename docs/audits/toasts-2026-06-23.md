## Convención de copy (es-MX)

- Verbo en pasado + sustantivo: `"Reserva creada"`, `"Pago registrado"`.
- Evitar `"exitosamente"`. Si todo salió bien, el toast verde ya lo comunica.
- Incluir folio/ID cuando exista: `"Factura FAC-0001 creada"`.
- Para validaciones de formulario usar `notifyValidation`, no `notifyError`.
- Para errores recuperables (saldo cero, duplicado) usar `notifyError({ severity: 'warning' })`.

## Decisiones explícitas

- **No se migraron las 379 llamadas existentes de `notifyError` a `severity: warning`** — requiere análisis caso por caso. La opción ya está disponible; aplicación incremental.
- **No se aplicó `notifyAsync` a operaciones largas concretas (timbrado, etc.)** — cada flujo ya tiene UI de loading propio. Documentado como follow-up.
- **`toast.error` se mantiene en `ErrorDetailsDialog.tsx:27`** — irónico notificar fallo de copiar dentro del propio diálogo de error con otro reporte estructurado.
